Git Rank
Git je standardem pro sdílení zdrojového kódu a veřejně dostupné repositáře se zdrojovým kódem mají velkou popularitu. Jednotlivé veřejné commity představují zajímavý způsob, jak prezentovat jednotlivé softwarově inženýrské dovednosti jejich autora a umožňují ukázat dobré dovednosti a návyky.
Cílem práce je navrhnout, implementovat a otestovat řešení, které bude schopné ohodnotit veřejně dostupné Git repositáře. Práce bude cílit na detekci comitů pro určení běžných SWI metriky v Git repositářích s využitím dostupných existujících nástrojů. Práce by měla umět pracovat s repositáři s více autory a umět vhodně popsat kontribuci podle technologií jako je Java, C# a Python. Pro získání seznamu repositářů je vhodné využít GitHub API.
Dílčí úkoly:
1) Proveďte analýzu systému správy verzí Git a rozhodněte, která uživatelská data z repositářů a jakým způsobem bude vhodné zpracovávat a dále analyzovat. 
2) Proveďte analýzu dostupných nástrojů pro statickou analýzu kódu pro Javu, C# a Python.
3) Navrhněte jednoduchou aplikaci, která bude získávat veřejné Git repositáře (z platformy GitHub) pro zadaného uživatele a provádět nad nimi analýzu dle navržených metrik.
4) Navrhněte přehledný způsob zobrazování výsledného reportu.
5) Vyhodnoťte daný report, Je patrné u uživatelů zlepšení v čase? Dosahují seniorní programátoři průměrně lepších výsledků?
